1. The AI-First Developer & The AMP Center

At Project D, we do not code line-by-line in isolated terminal windows. We orchestrate architectures directly within the AMP Center platform. The AMP Center is our primary assistant agentic workforce—replacing standalone coding tools (like Antigravity or Claude Code) by combining complex agentic pipelines with an organized, visual CRM UI. As a developer, your job is to build custom agent sandboxes and map API gateways using our integrated workspace:

CRM-Integrated Agent Workspace

The AMP Center acts as the central coordinator. Rather than managing raw agent outputs via CLIs, developers deploy specialized agents straight to client Kanban boards, where users see, track, and manage their executions visually.

Custom Agent Sandboxes

We co-engineer custom CRM views, database schema, and webhook configurations for each enterprise customer, wrapping raw agentic power inside a structured, user-friendly control center.

Technical Standard: If a task takes you more than 15 minutes of manual script writing, you aren't leveraging the AMP Center's developer-agent capabilities. Your role is to configure hooks, test pathways, and design workflows that keep the agentic workforce organized and efficient.

2. The Lead Dev's Secret Weapon

The Lead App & Web Developer sits in on Discovery calls to perform what we call "Real-Time Architecture", utilizing the developer panels inside the Settings and Strategic Planning modules:

The "Wow" Moment Execution

  • 01. Listen: During the first 15 minutes of the call, identify the prospect's visual taste, colors, and operational bottlenecks (e.g. driver callouts or slow lead responder times).
  • 02. Configure Developer Settings: While they are talking, use the developer panels in Settings to quickly provision a fresh database instance and scaffold a high-end landing page configured with their brand colors and logo.
  • 03. Map the Strategic Roadmap: Populate initial workflow cards in Strategic Planning to show them their custom agent blueprints.
  • 04. The Reveal: Share your screen. Showcase the live landing page and the CRM Leads Board containing their brand information. Demonstrate that we can build in 15 minutes what other agencies pitch in 3 weeks, proving the visual organizational power of our CRM UI.

3. Settings Channel Architecture

Developers are responsible for scripting webhook triggers, gateway endpoints, and API mappings inside the Settings console:

1. API & Webhook Scripting

Configure script listeners in the Settings developer console to process incoming payload variables (like SMS text messages or form submissions):

  • Script conditional statements to parse text keywords (e.g. `BOOK`, `TIRE`, `GLASS`).
  • Connect API responses to execute database state updates (e.g., changing lead stages to Closed Won or Demo Scheduled on the User Pipeline).

2. Workflow Connections

Link automated triggers to external services inside Settings to let agents coordinate tasks:

  • Twilio SMS Gateways: Wire up SMS webhooks to trigger text responses to standby arrays.
  • Database API Mappings: Connect custom databases to sync customer records with lead files.
  • Gateway Triggers: Listen for POS system checkout payloads to execute customer review booster campaigns.

4. Junior Developer Onboarding

As a Junior Developer, you manage the Rapid Implementation phase (Phase 03) to configure client settings and calendar slots:

  • Settings Setup & Provisioning: Provision database instances and generate API access credentials inside the Settings console for all onboarding clients.
  • Tactical Calendar Setup: Configure scheduling calendars, map time-slots, and integrate booking forms onto the client's domains.
  • Form Integrations: Embed review sliders and lead capture fields onto client web systems, ensuring submission data maps directly to their CRM board.
